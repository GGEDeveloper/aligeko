## Como Usar Esta Documentação

### Desenvolvedores Novos
1. Comece pela [Visão Geral da Arquitetura](./architecture/customer-area.md)
2. Consulte o [Guia de Estilo](./style-guide.md) para padrões de código
3. Verifique os exemplos de componentes e padrões

### Desenvolvedores Existentes
1. Atualize a documentação ao fazer alterações significativas
2. Adicione novos padrões ao Guia de Estilo quando necessário
3. Mantenha os diagramas e exemplos atualizados

## Convenções de Documentação

### Arquivos Markdown
- Use títulos com níveis lógicos (## para seções, ### para subseções)
- Inclua exemplos de código quando apropriado
- Mantenha os links internos atualizados

### Imagens e Diagramas
- Armazene imagens no diretório `./images`
- Use ferramentas como Mermaid para diagramas
- Mantenha as imagens otimizadas para web

## Contribuindo

1. Crie um branch para suas alterações
2. Atualize a documentação de forma clara e concisa
3. Abra um Pull Request com uma descrição detalhada
4. Solicite revisão de pelo menos um desenvolvedor sênior
